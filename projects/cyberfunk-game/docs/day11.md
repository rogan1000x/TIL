# Day 11 - 게임 난이도 시스템 v2 (동적 난이도)

## 오늘 배운 것
- **시간 기반 난이도**: 게임 경과 시간에 따라 기본 난이도 상향
- **Date.now()**: 현재 시간을 밀리초 단위로 가져오기
- **콤보 + 시간 조합**: 두 개의 난이도 요소 동시에 관리
- **게임 밸런싱**: 너무 쉽지도, 어렵지도 않게 조정

## 오늘 한 것
### 난이도 시스템 v2 설계
기존: 콤보만으로 난이도 결정
개선: 콤보 + 시간 경과 조합

### 새로운 난이도 로직
시간 기반 기본 난이도:

0-30초: EASY
30-60초: NORMAL
60초 이후: HARD


콤보 기반 추가 난이도:


콤보 3-5: NORMAL으로 상향
콤보 6+: HARD로 상향

콤보 리셋 시 (단어 놓침):

HARD → NORMAL (한 단계만)
NORMAL → EASY (한 단계만)
EASY는 유지 (최소 난이도)


### 구현 내용
1. **gameStartTime** 변수: 게임 시작 시간 기록
2. **getGameElapsedSeconds()**: 게임 경과 시간 계산
3. **getDifficultyByCombo()** 함수 개선:
   - 시간에 따른 기본 난이도 계산
   - 콤보로 추가 난이도 상향
   
4. **previousDifficulty** 변수: 이전 난이도 추적
5. **updateDifficulty()** 함수 개선:
   - 콤보 리셋 시 1단계만 내려감
   - 급격한 난이도 변화 방지

## 게임 경험 개선
- ✅ 게임이 점점 어려워짐 (긴장감)
- ✅ 콤보 리셋 후에도 계속 도전적 (너무 쉽지 않음)
- ✅ 회복감 + 재도전 욕구
- ✅ 자연스러운 난이도 곡선

## 막혔던 것
- 획일적인 난이도 (콤보만으로 결정)
  → 시간 기반 난이도 추가로 해결
  
- 콤보 리셋 시 너무 떨어짐
  → 1단계만 내려가도록 수정
  
- 모드 분리(초보자/열혈) 고민
  → 현재 시스템 개선으로 해결 (더 나음)
